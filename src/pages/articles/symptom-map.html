Bạn đang gặp gì? — chậm, RAM, double, view sai, consumer kẹt

PREVIEW · 2026-07-21 · ~6 phút đọc

Preview Bản nháp chờ review — title, nội dung, link có thể sửa trước khi bỏ nhãn.

Index xếp theo chủ đề học (concurrency → queue → memory). Lúc 2h sáng bạn nghĩ theo triệu chứng: chậm, OOM, double charge, dashboard sai, consumer kẹt.

Trang này là cửa vào: chọn nỗi → vài bài đủ để hỏi đúng câu và làm bước đầu. Không thay postmortem; thay “không biết bấm link nào”.

Checklist dài hơn cho latency: playbook service chậm (preview).

0. Mười giây — tách lớp

Chậm / đơ     → chờ I/O, kẹt loop, pool, dependency, retry?
RAM / OOM     → in-flight inventory, leak, cgroup?
Sai tiền / 2× → idempotency, redelivery, outbox?
Data “sai”    → stale / missing / wrong / uncertain?
Queue kẹt     → poison, unblock, lag, purpose channel?
Dependency    → breaker, bulkhead, shed?

Service / API chậm, p99 nổ

  1. Chờ hay tính? Deal vs do, block/non-block, event loop (sync ôm loop?).
  2. Deadline? Timeout budget, cancellation (việc mồ côi?).
  3. Dependency? Breaker, retry + jitter, retry cái gì.
  4. Pool DB? Connection pool hết chỗ — chậm mà CPU rảnh.
  5. Inventory? In-flight inventory, pool.

RAM tăng, OOM, process bị kill

  • Bao nhiêu việc đang bay? Unbounded in-flight, inventory, coroutine (rẻ ≠ vô hạn).
  • Số RAM nào? VSS/RSS/USS, heap vs RSS vs cgroup, GC.
  • Backpressure / shed: backpressure, load shedding.

Double charge / side-effect lặp

  • Idempotency, key scope.
  • Queue redelivery: lease / visibility, exactly-once thực dụng.
  • DB + publish: outbox, dual-write.
  • Timeout rồi bắn lại: unknown / uncertain, uncertain.

View / dashboard sai — ghi vẫn 200

  • Phân loại: stale · thiếu · sai · chưa chắc (+ demo).
  • Purpose channel: write / read / shared.
  • Lag / order: consumer lag, order budget.

Consumer kẹt, lag tăng, một message chặn cả hàng

  • Unblock (+ demo) — nhả lane.
  • Poison / DLQ, DLQ triage.
  • Chọn strategy: recovery playbook, ignore khi nào đúng.
  • Resend cẩn: resend shared, shared multi-tenant.

Dependency chết / chậm kéo sập mình

  • Breaker, fail modes.
  • Bulkhead, load shedding.
  • Backpressure, rate limit vs pool.

Race / stampede

  • TOCTOU async.
  • Cache stampede, single-flight.

Cách dùng hub

  1. Chọn 1–2 triệu chứng gần nhất (đừng 5 cùng lúc).
  2. Đọc 1 bài “phân loại” + 1 bài “làm ngay”.
  3. Chỉ sau đó mới deep-dive pattern (breaker internals, saga…).

Một câu

Triệu chứng trước, tên pattern sau.
